# Security Policy

## Supported versions

Security fixes are provided on a best-effort basis for the latest stable
release and the active prerelease. Older releases may receive fixes at the
maintainer's discretion.

## Reporting a vulnerability

Report vulnerabilities through
[GitHub Issues](https://github.com/martinvidovic/opencode-limits/issues). This
project does not currently provide a private vulnerability-reporting channel
or a response-time guarantee.

All issue content is public. Never include access tokens, refresh tokens,
cookies, authorization headers, raw credential files, unredacted logs, or
other secrets. Revoke and rotate any credential that may have been exposed
before filing the report. Provide the smallest sanitized reproduction that
still explains the impact and affected versions.
