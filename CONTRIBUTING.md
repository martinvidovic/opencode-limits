# Contributing

Contributions to opencode-limits are welcome.

## Proposing changes

Small fixes and documentation improvements may be submitted directly as pull
requests. Open an issue and agree on the direction before working on:

- a new Provider Adapter;
- a public API change;
- credential access, refresh, storage, redaction, or endpoint behavior; or
- a substantial feature or architectural change.

This avoids substantial work on changes that may not fit the project.

## Pull requests

Keep pull requests focused and explain their user-visible behavior. Include or
update tests and documentation when applicable. All required automated checks
must pass before a pull request can be merged.

The maintainer has final authority over design, review, acceptance, and merge
decisions. Participation does not grant automatic merge rights or voting
authority.

## Provider Adapters

A Provider Adapter is an in-repository integration contributed upstream and
shipped as part of opencode-limits. New Provider Adapters must satisfy the
project's shared conformance and documentation requirements.

Once merged, a Provider Adapter is maintained by the project under its
best-effort support policy. Its original contributor is encouraged but not
obligated to continue maintaining it. The maintainer may fix, deprecate,
disable, or remove an adapter that becomes unsafe or unmaintainable.
